# Recording Guide: VibeFinder 2.0

Use this as a simple speaking guide for your Loom video.

---

## What The Platform Does

VibeFinder 2.0 takes a natural-language music request, finds matching songs, explains the recommendations, and shows reliability information.

Example prompt:

`Need calm music for late-night coding`

---

## Platform Structure: Top To Bottom

### 1. Top banner

- introduces the project
- highlights the main AI features

### 2. Quick scene buttons

Buttons:

- `Late-Night Focus`
- `Gym Surge`
- `Reflective Journaling`
- `Conflicted Edge`
- `Debugging at 1am`

What they do:

- load a preset query
- instantly run a demo example

### 3. Composer

This is the main control section.

It includes:

- prompt box
- `Mode`
- `Style`
- `Top K`
- `Diversity`
- `Multi-source`
- `Styled output`
- `Run Live Demo`

What they do:

- prompt box: enter the music request
- `Mode`: changes recommendation priority
- `Style`: changes explanation tone
- `Top K`: number of songs returned
- `Diversity`: reduces repetitive results
- `Multi-source`: turns extra retrieval on or off
- `Styled output`: turns specialized explanations on or off
- `Run Live Demo`: runs the full system

### 4. Summary strip

Shows:

- confidence
- support doc matches
- trace steps
- warnings
- active style

### 5. Recommendations

Shows:

- top song
- score
- explanation
- more recommended songs

### 6. Right-side system panels

These show:

- reliability status
- warnings
- how the request was interpreted
- agent trace steps

### 7. Retrieval section

Shows:

- support document evidence
- song evidence used in ranking

### 8. Bottom tabs

Tabs:

- `Benchmarks`
- `Evaluation`
- `Architecture`

What they do:

- `Benchmarks`: reliability results
- `Evaluation`: baseline vs enhanced comparison
- `Architecture`: system diagram

### 9. Sidebar

Contains:

- quick scene buttons
- `Run Reliability Benchmarks`
- `Run Optional Feature Evaluation`

What the buttons do:

- `Run Reliability Benchmarks`: runs reliability checks
- `Run Optional Feature Evaluation`: runs the feature comparison

---

## Simple Example Walkthrough

Use this prompt:

`Need calm music for late-night coding`

What to do:

1. Start at the top and quickly point out the main sections.
2. Go to the Composer.
3. Enter the prompt.
4. Click `Run Live Demo`.
5. Explain the results from top to bottom.

What to say:

“Here I’m entering a natural-language request for calm music for late-night coding. The system interprets the request, retrieves supporting evidence, ranks the songs, and explains why they match. The recommendation area shows the song results, the right-side panels show reliability and parser details, and the retrieval section shows the evidence behind the answer.”

---

## Short Closing Line

“The main idea of this platform is not just to recommend songs, but to show how the recommendation was made and whether the result looks reliable.”
